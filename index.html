Документация проекта local-recomendation-bot¶

local-recomendation-bot это бот для телеграмма, который рекомендует различные категории досуга на основании вашего местоположения и предыдущих выборов.

Возможности

  • Получение геолокации встроенным способом через телеграмм или через текстовый адресс.

  • Рекомендации по базе данных на основании расстояния

  • Возможность ставить оценки рекомендациям, которые будут учитываться при дальнейших рекомендациях

core.recommender.CandidatesHolder¶

class core.recommender.CandidatesHolder(type_to_candidates=None)¶

Хранилище кандидатов для рекомендации.

В класс можно передать уже созданный словарь type_to_candidates, либо оставить поле пустым, после чего будет автоматически создан словарь type_to_candidates для всех доступных на текущий момент видов рекомендаций.

Метод update обновит информацию о всех доступных в базе данных кандидатах, то есть перезапишет их для дальнейшего использования роекомендером.

core.recommender.EmbeddingsHolder¶

class core.recommender.EmbeddingsHolder(type_to_embeddings=None)¶

Хранилище эмбеддингов всех кандидатов.

В класс можно передать уже созданный словарь type_to_embeddings, либо оставить поле пустым, после чего будет автоматически создан словарь type_to_embeddings для всех доступных на текущий момент видов рекомендаций.

Метод update обновит информацию о всех доступных в базе данных эмбеддингах, то есть перезапишет их для дальнейшего использования роекомендером.

core.recommender.FeedbackEventProcessor¶

class core.recommender.FeedbackEventProcessor(history_path)¶

В этом классе происходит взаимодействие пользователей с рекомендациями.

Содержит метод write_user_item_rating(), который пишет оценку пользователя в файл history_path.

Содержит метод read_user_history, который возвращает словарь item_to_rating с историей оценок пользователя, или None, если истории нет.

clear_user_history(user_id)¶

Удаление пользовательской истории.

read_user_history(user_id, limit=None)¶

Читаем историю оценок пользователя.

Возвращает словарь item_id_to_rating, который содержит последние limit оценок пользователя. Если limit не задан, то возвращает всю историю без ограничений.

write_user_item_rating(user_id, item_id, rating_good)¶

Записывает оценку пользователя элементу рекомендаций.

Запись происходит в формате type,id,rating, где type и id - есть поля ItemId, а rating - плюс или минус единица в зависимости от оценки пользователя.

core.recommender.Recommender¶

class core.recommender.Recommender(item_type, candidates_holder, embeddings_holder=None, feedback_event_processor=None)¶

Основной класс рекомендера с единой для всех вертикалей логикой.

before_recommend(USER_INFO, user_history_limit=None)¶

Подготовка к вызову рекомендера.

Метод для подготовки данных перед вызовом рекомендера для его корректной работы:

Создаёт множество recommend_history, если история пользователя отсутсвует. Записывает в item_id_to_rating последние user_history_limit оценок пользователя. Если user_history_limit не задано, то запишет всю историю пользователя.

get_heavy_recommender_items(USER_INFO, light_recommender_items, embeddings, limit)¶

Тяжёлый рекомендер.

Возвращает список из limit рекомендательных карточек, отранжированных по косиносному расстоянию пользовательских и айтемных эмбеддингов Диего. Результат тяжёлого рекомендера будет обработан в stream_blender.

get_light_recommender_items(USER_INFO, candidates, coords, limit)¶

Лёгкий рекомендер.

Возвращает список из limit рекомендательных карточек, отранжированных по расстоянию, которые затем будут переданы в тяжёлый рекомендер.

recommend(USER_INFO, user_history_limit=None, light_recommender_limit=200, heavy_recommender_limit=20, blender_limit=5)¶

Основная функция рекомендера.

Рекомендер возвращает список из limit карточек рекомендаций, которые затем будут обработаны в большом блендере

stream_blender(USER_INFO, recommended_items, blender_limit=5, mode='Diego', temperature=1)¶

Возвращает финальную пачку рекомендаций с вертикали.

Получает карточки лёгкого рекомендера, перемешивает их согласно определённым правилам, которые задаются в mode, и возвращает финальные blender_limit карточек рекомендаций.

По умолчанию выбран mode=“Diego“, который позволяет получать разнообразные по местоположению рекомендации. Параметр temperature задаёт степень разнообразия (чем больше temperature, тем его меньше)
